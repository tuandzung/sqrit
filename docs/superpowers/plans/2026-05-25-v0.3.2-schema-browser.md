# v0.3.2 — Schema browser Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand the schema browser to include every object kind each backend supports — indexes, triggers, functions, procedures, sequences, materialized views — under a namespace-aware tree (PostgreSQL schemas / MySQL databases / SQLite single implicit namespace). Document as ADR 8. Ship as v0.3.2.

**Architecture:** `Database::schema_info()` returns `Vec<Namespace>` instead of flat `tables`/`views`. Each adapter fills only the buckets the backend supports; missing kinds stay empty. Explorer `TreeItem` becomes hierarchical: `Namespace → Group(kind) → Object → Column`. Single-namespace cases (SQLite, single-DB MySQL) auto-collapse the namespace layer. System schemas are filtered server-side. Per-adapter quoting helper builds `SELECT * FROM "ns"."obj" LIMIT 100`.

**Tech Stack:** Rust 2021, ratatui 0.29, sqlx 0.8 (PG/MySQL), rusqlite 0.32 (SQLite), tokio (existing).

    View,

    Trigger,

    Sequence,

            ObjectKind::Procedure => "Procedures",
            ObjectKind::Sequence => "Sequences",
        }

//! Adapter-specific identifier quoting. Used when building qualified
//! `SELECT * FROM <ns>.<obj>` queries from explorer selections so backtick
//! / double-quote rules are respected.
//!
//! Embedded quote chars are doubled (SQL standard). Anything else is treated
//! as a normal identifier character — no length validation, no reserved-word
//! check.

pub fn quote_pg(ident: &str) -> String {
    let escaped = ident.replace('"', "\"\"");
    format!("\"{}\"", escaped)
}

pub fn quote_mysql(ident: &str) -> String {
    let escaped = ident.replace('`', "``");
    format!("`{}`", escaped)
}

pub fn quote_sqlite(ident: &str) -> String {
    // SQLite supports double-quotes for identifiers (preferred over `[name]`
    // and backticks for portability).
    let escaped = ident.replace('"', "\"\"");
    format!("\"{}\"", escaped)
}
```

async fn list_indexes_sqlite(&self) -> anyhow::Result<Vec<IndexObject>> {
    let result = self
        .execute(
            "SELECT name, tbl_name, sql FROM sqlite_master
             WHERE type='index' AND name NOT LIKE 'sqlite_%'
             ORDER BY name",
        )
        .await?;
    Ok(result
        .rows
        .iter()
        .filter_map(|r| {
            let name = match r.get("name") { Some(Value::Text(s)) => s.clone(), _ => return None };
            let table = match r.get("tbl_name") { Some(Value::Text(s)) => s.clone(), _ => return None };
            let sql = match r.get("sql") { Some(Value::Text(s)) => s.clone(), _ => String::new() };
            let unique = sql.to_uppercase().contains("UNIQUE");
            Some(IndexObject { name, table, unique })
        })
        .collect())
}

async fn list_triggers_sqlite(&self) -> anyhow::Result<Vec<TriggerObject>> {
    let result = self
        .execute(
            "SELECT name, tbl_name, sql FROM sqlite_master
             WHERE type='trigger'
             ORDER BY name",
        )
        .await?;
    Ok(result
        .rows
        .iter()
        .filter_map(|r| {
            let name = match r.get("name") { Some(Value::Text(s)) => s.clone(), _ => return None };
            let table = match r.get("tbl_name") { Some(Value::Text(s)) => s.clone(), _ => return None };
            let sql = match r.get("sql") { Some(Value::Text(s)) => s.clone(), _ => String::new() };
            let upper = sql.to_uppercase();
            let event = if upper.contains(" INSERT ") {
                "INSERT".to_string()
            } else if upper.contains(" UPDATE ") {
                "UPDATE".to_string()
            } else if upper.contains(" DELETE ") {
                "DELETE".to_string()
            } else {
                String::new()
            };
            Some(TriggerObject { name, table, event })
        })
        .collect())
}
```

    table: &str,

    let rows = sqlx::query_as::<_, (String, String, bool, bool)>(
        "SELECT column_name, data_type, is_nullable = 'YES', EXISTS (
            SELECT 1 FROM information_schema.table_constraints tc
            JOIN information_schema.key_column_usage kcu
                ON tc.constraint_name = kcu.constraint_name
                AND tc.table_schema = kcu.table_schema
            WHERE tc.table_name = $2 AND tc.table_schema = $1
              AND tc.constraint_type = 'PRIMARY KEY'
              AND kcu.column_name = c.column_name

        FROM information_schema.columns c
        WHERE table_schema = $1 AND table_name = $2
        ORDER BY ordinal_position",
    )
    .bind(schema)
    .bind(table)
    .fetch_all(pool)
    .await?;

    Ok(rows
        .into_iter()
        .map(|(name, data_type, nullable, is_primary_key)| ColumnInfo {
            name,
            data_type,
            nullable,
            is_primary_key,

async fn schema_info(&self) -> anyhow::Result<SchemaInfo> {
    use super::types::{Namespace, ViewObject};

    let schemas = self.list_user_schemas_pg().await?;
    let mut namespaces = Vec::with_capacity(schemas.len());
    for schema in schemas {
        let table_names = self.list_tables_in_schema_pg(&schema).await?;
        let view_names = self.list_views_in_schema_pg(&schema).await?;
        let matview_names = self.list_matviews_in_schema_pg(&schema).await?;

        let mut tables = Vec::new();
        for name in &table_names {
            let columns = self.list_columns_qualified_pg(&schema, name).await?;
            tables.push(TableObject { name: name.clone(), columns });
        }
        let mut views = Vec::new();
        for name in &view_names {
            let columns = self.list_columns_qualified_pg(&schema, name).await?;
            views.push(ViewObject { name: name.clone(), columns });
        }
        let mut matviews = Vec::new();
        for name in &matview_names {
            let columns = self.list_columns_qualified_pg(&schema, name).await?;
            matviews.push(ViewObject { name: name.clone(), columns });
        }
        let indexes = self.list_indexes_in_schema_pg(&schema).await?;
        let triggers = self.list_triggers_in_schema_pg(&schema).await?;
        let functions = self.list_routines_in_schema_pg(&schema, 'f').await?;
        let procedures = self.list_routines_in_schema_pg(&schema, 'p').await?;
        let sequences = self.list_sequences_in_schema_pg(&schema).await?;

        namespaces.push(Namespace {
            name: schema,
            tables,
            views,
            materialized_views: matviews,
            indexes,
            triggers,
            functions,
            procedures,
            sequences,
        });
    }
    Ok(SchemaInfo { namespaces })
}

## Task 15: ADR 8

**Files:**
- Create: `docs/adr/0008-namespace-aware-introspection.md`

- [ ] **Step 1: Write the ADR**

```markdown
# ADR 8 — Namespace-aware introspection

**Status**: accepted
**Date**: 2026-MM-DD (commit date)

## Context

v0.2 `SchemaInfo` was flat: `{ tables: Vec<TableInfo>, views: Vec<ViewInfo> }`. This collapsed multi-schema PostgreSQL into a single bucket (only `public` was queried) and left no place to surface indexes, triggers, functions, procedures, sequences, or materialized views. v0.3.2 plans to surface all of these.

## Decision

`Database::schema_info()` returns a namespace-rooted result:

```
SchemaInfo { namespaces: Vec<Namespace> }
Namespace { name, tables, views, materialized_views, indexes, triggers, functions, procedures, sequences }
```

- SQLite: one namespace with empty `name` (`""`); never has procedures/functions/sequences/matviews.
- PostgreSQL: one namespace per user-schema (system schemas — `pg_catalog`, `information_schema`, `pg_toast`, `pg_temp_*` — filtered server-side).
- MySQL: one namespace whose `name` is the result of `DATABASE()` (the connected database).

Explorer renders Namespace → Group(kind) → Object → Column. Single-namespace case auto-collapses the namespace row (SQLite, single-DB MySQL).

## Alternatives considered

- **Flat with `kind` tag.** Rejected: render explosion on DBs with hundreds of indexes; no natural collapse boundary.
- **Per-kind separate trait methods (`list_indexes`, `list_triggers`, …).** Rejected: N adapter round-trips per Explorer open; harder to keep introspection atomic; harder to extend without churning the trait.

## Consequences

- **Invariant V11** (new in CLAUDE.md): `Database::schema_info()` returns a namespace-rooted result; never flat.
- Each adapter pays a fixed introspection cost on connect (one query per kind per namespace). Acceptable for typical DBs; lazy per-kind load is a future ADR 8 amendment if perf complaints arrive.
- A future env flag `SQRIT_DISABLE_INTROSPECTION_KINDS` is the emergency rollback if a particular kind is unexpectedly slow on a real DB.
- All explorer-driven `SELECT *` queries now go through `db::quote` and produce qualified `"<ns>"."<obj>"` (PG/SQLite) or `` `<ns>`.`<obj>` `` (MySQL); SQLite drops the prefix when `ns` is empty.
```
