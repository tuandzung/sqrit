# v0.3.1 — Hint bar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a single-row, mode-aware keybinding hint bar above the status bar. Left half renders the active mode's top bindings (from `ModeHandler::bindings()`); right half always shows the constant `<sp> cmd  ? help` suffix. Truncates gracefully on narrow terminals. Toggleable via `~/.sqrit/config.toml`. Document as ADR 7. Ship as v0.3.1.

**Architecture:** Extract layout split + status rendering in `app.rs::render_main` so the hint row can slot between the content area and the status row. New `src/hint_bar.rs` owns layout math (compose, measure, truncate). Hint content sourced from each mode's existing `bindings()` (no new field; the convention is "most-important-first"). Optional `[hint_bar]` TOML section on both `AppConfig` and `Theme`; missing fields fall back to existing palette colors. Config read at startup — no runtime toggle in v0.3.1.

**Tech Stack:** Rust 2021, ratatui 0.29, crossterm 0.28, tokio (existing).

```rust
//! Hint bar — single row of mode-aware keybindings above the status bar.
//!
//! Layout: mode bindings left, fixed palette suffix right, separator pipe
//! between them when both fit. Truncates trailing mode bindings first.

use ratatui::layout::Rect;
use ratatui::style::{Modifier, Style};
use ratatui::text::{Line, Span};
use ratatui::widgets::Paragraph;
use ratatui::Frame;

use crate::mode::KeyBinding;
use crate::theme::Theme;

pub const PALETTE_SUFFIX: &str = "<sp> cmd  ? help";

/// Minimum width below which the row is rendered empty (or omitted entirely
/// when `auto_hide_narrow = true`). Picked so even one mode binding usually
/// fits.
pub const MIN_WIDTH: u16 = 40;

/// Result of laying out the hint row. `width_used` is the columns actually
/// consumed by the spans (excluding the gap fill). Useful for tests.
pub struct Layout {
    pub left: Vec<Span<'static>>,
    pub right: Vec<Span<'static>>,
    pub gap: u16,
}

/// Pure layout pass: produce left + right span vectors for the given mode
/// bindings, the constant palette suffix, and the row width. `theme` only
/// supplies the colors; no Frame interaction.
///
/// Algorithm:
/// 1. Render `left` = all mode bindings as `"<key> <label>"` separated by 2 spaces.
/// 2. Render `right` = `PALETTE_SUFFIX`.
/// 3. If `left + " │ " + right <= width` → keep both, separator pipe.
/// 4. Else if `left <= width` → keep left only, no right.
/// 5. Else drop trailing mode bindings one at a time until it fits.
/// 6. If even one binding is too wide → render `…` only.
pub fn compose(bindings: &[KeyBinding], theme: &Theme, width: u16) -> Layout {
    if width == 0 {
        return Layout {
            left: vec![],
            right: vec![],
            gap: 0,
        };
    }
    let key_style = Style::default()
        .fg(theme.hint_bar_key)
        .add_modifier(Modifier::BOLD);
    let body_style = Style::default().fg(theme.hint_bar_fg);
    let sep_style = Style::default().fg(theme.hint_bar_separator);

    let right_text = PALETTE_SUFFIX;
    let right_w = right_text.chars().count() as u16;
    let sep_w = 3u16; // " │ "

    // Try decreasing prefix of `bindings` to find the largest set that fits.
    for take in (0..=bindings.len()).rev() {
        let slice = &bindings[..take];
        let left_text = render_left_text(slice);
        let left_w = left_text.chars().count() as u16;

        // Try with-suffix first.
        if take == bindings.len() && left_w + sep_w + right_w <= width {
            return Layout {
                left: spans_for_bindings(slice, key_style, body_style),
                right: vec![
                    Span::styled(" │ ".to_string(), sep_style),
                    Span::styled(right_text.to_string(), body_style),
                ],
                gap: width - left_w - sep_w - right_w,
            };
        }
        if left_w + sep_w + right_w <= width {
            return Layout {
                left: spans_for_bindings(slice, key_style, body_style),
                right: vec![
                    Span::styled(" │ ".to_string(), sep_style),
                    Span::styled(right_text.to_string(), body_style),
                ],
                gap: width - left_w - sep_w - right_w,
            };
        }
        if left_w <= width {
            // Drop the suffix; keep as many mode bindings as fit.
            return Layout {
                left: spans_for_bindings(slice, key_style, body_style),
                right: vec![],
                gap: width - left_w,
            };
        }
    }

    Layout {
        left: vec![Span::styled("…".to_string(), body_style)],
        right: vec![],
        gap: width.saturating_sub(1),
    }
}

fn render_left_text(bindings: &[KeyBinding]) -> String {
    let mut out = String::new();
    for (i, b) in bindings.iter().enumerate() {
        if i > 0 {
            out.push_str("  ");
        }
        out.push_str(b.key);
        out.push(' ');
        out.push_str(b.action);
    }
    out
}

fn spans_for_bindings(
    bindings: &[KeyBinding],
    key_style: Style,
    body_style: Style,
) -> Vec<Span<'static>> {
    let mut out = Vec::new();
    for (i, b) in bindings.iter().enumerate() {
        if i > 0 {
            out.push(Span::styled("  ".to_string(), body_style));
        }
        out.push(Span::styled(b.key.to_string(), key_style));
        out.push(Span::styled(" ".to_string(), body_style));
        out.push(Span::styled(b.action.to_string(), body_style));
    }
    out
}

pub fn render(frame: &mut Frame, area: Rect, bindings: &[KeyBinding], theme: &Theme) {
    if area.width == 0 || area.height == 0 {
        return;
    }
    let layout = compose(bindings, theme, area.width);
    let mut line_spans = layout.left;
    if !layout.right.is_empty() {
        if layout.gap > 0 {
            line_spans.push(Span::raw(" ".repeat(layout.gap as usize)));
        }
        line_spans.extend(layout.right);
    }
    let p = Paragraph::new(Line::from(line_spans))
        .style(Style::default().bg(theme.hint_bar_bg));
    frame.render_widget(p, area);
}
```

## Task 10: ADR 7

**Files:**
- Create: `docs/adr/0007-hint-bar.md`

- [ ] **Step 1: Write the ADR**

```markdown
# ADR 7 — Hint bar layout

**Status**: accepted
**Date**: 2026-MM-DD (set to commit date)

## Context

`?` opens a help overlay listing the active mode's keybindings (since v0.2). The overlay is on-demand and modal — it blocks the rest of the UI while open. Discoverability of mode keys remains low: a newcomer in Results doesn't know `v` opens the cell viewer or `/` opens the filter until they press `?`. lazygit solves this with an always-visible hint row at the bottom of the screen.

## Decision

Add a single reserved row directly above the status bar that renders mode-aware keybinding hints.

- **Layout (Section B from brainstorming):** extra row above status, not replacing it. Status bar keeps its full width for error messages.
- **Content (Section C from brainstorming):** hybrid — mode bindings left-justified (sourced from `ModeHandler::bindings()`), constant palette suffix `<sp> cmd  ? help` right-justified, separator pipe between them.
- **Toggleable:** `[hint_bar].enabled` in `~/.sqrit/config.toml` (default `true`), plus `auto_hide_narrow` for very small terminals.
- **Theme integration:** optional `hint_bar_*` colors in `themes/*.toml`; missing fields fall back to existing palette colors.

## Alternatives considered

- **Replace status bar (Section A).** Rejected: error message width gets squeezed unpredictably; mode-specific hints rotate in and out of the same row as error text, which is visually noisy and risks overwriting status during long error sequences.
- **`<space>`-armed popup only (Section C of original).** Rejected: diverges from the "always-visible" goal that motivates the feature; help overlay already covers on-demand surfacing.

## Consequences

- **Invariant V10** (new in CLAUDE.md): hint bar bindings come from `ModeHandler::bindings()` only — never inline strings. Same source as help overlay ensures the two never drift.
- Every new mode handler must order `bindings()` most-important-first; PR review catches violations.
- One terminal row reserved (or zero when disabled / auto-hidden narrow).
- Theme schema is forward-additive: pre-v0.3.1 user TOMLs without the `hint_bar_*` keys still render correctly via per-field fallback.
```
